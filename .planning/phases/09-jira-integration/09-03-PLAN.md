**IMPORTANT:** The existing `SaveJiraCredentials` handler method body must be replaced with a stub that returns 410 Gone with a plain-text message "replaced by /app/settings/jira/connections in Phase 9" — do NOT delete the method yet (routes.go still references it). The route will be replaced in routes.go.

Add four new handler methods to `handler.go`:

1. `CreateJiraConnection(w http.ResponseWriter, r *http.Request)`:
   - Parse form: display_name, base_url, email, token (all required; return 422 HTML fragment if any missing)
   - If jiraClientFactory is nil: return 422 "Jira integration not configured"
   - Build `model.JiraConnection{DisplayName, BaseURL, Email, Token}`
   - Validate by calling `jiraClientFactory(conn).Ping(r.Context())`; on error return 422 with error message HTML fragment
   - On success: call `jiraConnStore.Create(r.Context(), conn)`
   - Return 200 with updated Jira connection list HTML fragment (call `renderJiraConnectionList`)
   - On validation failure `ErrJiraUnauthorized`: message "Invalid credentials — check email and API token"
   - On validation failure `ErrJiraUnavailable`: message "Could not reach Jira instance — check base URL"

2. `DeleteJiraConnection(w http.ResponseWriter, r *http.Request)`:
   - Parse `{id}` path param as int64; return 400 on parse error
   - Call `jiraConnStore.Delete(r.Context(), id)`
   - Return 200 with updated connection list HTML fragment

3. `SetDefaultJiraConnection(w http.ResponseWriter, r *http.Request)`:
   - Parse `{id}` path param as int64
   - Call `jiraConnStore.SetDefault(r.Context(), id)`
   - Return 200 with updated connection list HTML fragment

4. `SaveJiraRepoMapping(w http.ResponseWriter, r *http.Request)`:
   - Parse form: repo_full_name (string), jira_connection_id (int64; 0 means clear)
   - Call `jiraConnStore.SetRepoMapping(r.Context(), repoFullName, connectionID)`
   - Return 200 with empty 200 response (the popover re-renders via the repo list OOB swap on success — or a simple success fragment)

Add private helper `renderJiraConnectionList(w http.ResponseWriter, r *http.Request)` that:
- Calls `jiraConnStore.List(r.Context())`, maps to `[]vm.JiraConnectionViewModel`
- Renders the `components.JiraConnectionList(conns)` templ component to w
